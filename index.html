Free and Open Source media management

ACTIVAE 2.0 is a web-based Media Asset Management System. It's scalable, open source, written in Python, designed to run on Ubuntu Server, and very powerful in its ability to read, manipulate, & create multiple versions of media.
ACTIVAE 2.0 supports live format transcoding, workflow, multiple users with different roles and ALCs; It uses standard metadata. Also, it can be use as a video hosting platform to playback online videos, sounds and images.
It was originally developed by CENATIC, the Spanish National Open Source Center. The project was discontinued by CENATIC, but a really nice guy recovered the source code to give it a second opportunity.
It is now being translated to English so it's finally ready for prime time.

Why even bother with media management, isn't it free with cloud storage?

Actually no. Not even close. As more organizations, commercial & non-profit alike, generate more media, it becomes part of their value chain, their know-how and production processes. Yet over time the major commercial cloud storage services [Google Drive, Dropbox, etc.] have become less capable of cataloging and retrieving complex, essential metadata. Worse still, there is no commercial cloud storage provider that allows users to edit their own standard metadata.

ACTIVAE 2.0 is developed under the guiding concept that intelligent media management sees data not just as a single media file (audio, video, or image) with fixed metadata, but as the combination of that file, its metadata, and all necessary sidecar data [i.e.; closed captioning files, album art, etc]- plus the ability to edit all of them as needed.

This allows for users throughout the organization to analyze, document, and most importantly, manipulate (read: make important changes to) data in large batches, quickly. In other words, don't scroll, click, and drag one or two files at a time when you can search, sort, and filter in large meaningful groups.

About ACTIVAE 2.0

Most organizations are seeing an increased need for platforms that allow complex management of multimedia assets. And there are now several commercial platforms that can do this: Canto Cumulus, Artesia DAM, Fuzion Media Asset Management, Nuexeo DAM, and others.
However, the number of simple and scalable platforms based on open source software was sorely lacking for a long time.
ACTIVAE 2.0 fills this need.

This fork, ACTIVAE 2.0, is now being actively developed, translated to English, and receiving a massive UI overhaul.

Why the renaming, translation, and rebranding? Because in this day and age, no user should be forced to deal with a confusing GUI. This is also our gift to the FOSS developer community.

Distinguishing features:

  • Scalability: ACTIVAE 2.0 consists of four modules: transcoding, data access, web frontend, and an API exported via XML/RPC. Each one of them allows load balancing and high availability, so the scalability possibilities are broad:
  • It can all run on one machine or additional machines can be arranged as required.
  • If the use of the platform requires more storage, you can easily hot-plug it.
  • If you do a lot of transcoding of multimedia formats, machines can be dedicated specifically for these jobs and even more can be added as you see the need.
  • Load balancing: is provided by the internal engine of the platform, the Cherokee web server. Access to the application can be configured in this way to provide service in those cases where visitors to the platform are abundant.
  • Transcoding: ACTIVAE 2.0 allows advanced format transcoding in the platform itself- including a roadmap for popular film/video production formats, such as DNxHD & CineForm. If a production format video is added (like DNxHD, CineForm, or ProRes), it can be rendered quickly and quietly into an H.264 MP4 or WebM video for publishing on the internet. You can even convert a video to an MP3; ACTIVAE 2.0 will automatically strip the video layer and extract the audio. It supports transcoding between 200 different formats/containers, and most importantly, the transcoding is done asynchronously - decoupled and detached (in the background) while you can continue working with ACTIVAE 2.0. It is a non-blocking process... think about all the media production time you'll get back when your workstation stops holding you back during render times!
  • 100% AJAX GUI: provided by the CTK library Cherokee Tool Kit , a jQuery wrapper written in Python that allows the development of advanced, light and professional user interfaces.
  • Written in Python: The platform is fully developed in Python 2.5, which should be approachable and familiar to many would-be contributors: the API that is exported via XML/RPC, the AJAX user interface, the control and business logic, the database access, etc.
  • It is open source: ACTIVAE 2.0 continues under the New BSD License. Imagine the possibilities...
  • It is maintained by media production professionals for media production professionals: As new codecs and media formats become available, you can bet that the open source implementations will be included in ACTIVAE 2.0!

Activae uses the Cherokee Web Server and includes an advanced API so is well suited for integration (but XML/RPC rather than the more conventional REST or CMIS gateways). It includes a separate transcoding server detached from the web application which permits it to scale quite easily.
