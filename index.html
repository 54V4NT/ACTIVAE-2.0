Free and Open Source media management

ACTIVAE 2.0 is a web Media Asset Management System. It's open source and was developed originally by CENATIC, the Spanish National Open Source Center. The project was discontinued by CENATIC, but a really nice guy recovered the source code to give it a second opportunity. I'm managing development of this project, translating the documentation to English, and maintaining it temporarily until somebody can join/replace me. ACTIVAE 2.0 supports live format transcoding, workflow, multiple users with different roles and ALCs; It uses standard metadata. Also, it can be use as a video hosting platform to playback online videos, sounds and images.

Why even bother with media management, isn't it free with cloud storage?

Actually no. As more businesses, organizations, and institutions of all kinds generate more media that becomes part of their value chain, their know-how and production processes, the major commercial cloud storage services [Google Drive, Dropbox, etc.] seem to have become less capable of cataloguing and retrieving complex, essential metadata.

To us intelligent media management means the system sees data not just as a single media file (audio, video, or image) with fixed metadata, but as the combination of that file, its metadata, and all necessary sidecar data [i.e.; closed captioning files, album art, etc]- and the ability to edit all of them.

This allows for the entire organization to analyze, document, and most importantly, manipulate (read: make important changes to) data in large batches, quickly. In other words, don't scroll, click, and drag one or two files at a time when you can search, sort, and filter in large meaningful groups.

About ACTIVAE 2.0

Most organizations feel the need for platforms that allow the complex management of multimedia assets. There are several platforms that can do that: Canto Cumulus, Artesia DAM, Fuzion Media Asset Management, Nuexeo DAM, and others.
However, the number of simple and scalable platforms based on open source software is sorely lacking and that is why CENATIC developed the original Activae platform for multimedia asset management: it was highly scalable, open source, vendor-independent, fully developed in Python, and designed for the Ubuntu Server.

The active development fork, ACTIVAE 2.0, is now being developed independently, exclusively in English, and with a massive UI overhaul.

Why the renaming/rebranding? Seriously, why not? This project was close to biting the dust. The least we can do is give it a little bit of a face-lift.

Distinguishing features:

  • Scalability: ACTIVAE 2.0 consists of four modules: transcoding, data access, web frontend, and an API exported via XML/RPC. Each one of them allows load balancing and high availability, so the scalability possibilities are broad:
  • It can all run on one machine or additional machines can be arranged as required.
  • If the use of the platform requires more storage, you can easily hot-plug it.
  • If you do a lot of transcoding of multimedia formats, machines can be dedicated specifically for these jobs and even more can be added as you see the need.
  • Load balancing: is provided by the internal engine of the platform, the Cherokee web server [5]. Access to the application can be configured in this way to provide service in those cases where visitors to the platform are abundant.
  • Transcoding: ACTIVAE 2.0 allows advanced format transcoding in the platform itself- including a roadmap for popular film/video production formats, such as DNxHD & CineForm. If a production format video is added (like DNxHD, CineForm, or ProRes), it can be rendered quickly and quietly into an H.264 MP4 or WebM video for publishing on the internet. You can even convert a video to an MP3; ACTIVAE 2.0 will automatically strip the video layer and extract the audio. It supports transcoding between 200 different formats/containers, and most importantly, the transcoding is done asynchronously - decoupled and detached (in the background) while you can continue working with ACTIVAE 2.0. It is a non-blocking process... think about all the media production time you'll get back when your workstation stops holding you back during render times!
  • 100% AJAX GUI: provided by the CTK library Cherokee Tool Kit , a jQuery wrapper written in Python that allows the development of advanced, light and professional user interfaces.
  • Pythonicness: The platform is fully developed in Python 2.5, familiar to many readers: the API that is exported via XML/RPC, the AJAX user interface, the control and business logic, the database access, etc.
  • It is open source: ACTIVAE 2.0 is licensed under the New BSD License terms. This allows modification, use, distribution, sale, or relicensing, which is an advantage to any individual, company, or organization that wishes to use their properties for personal use or to provide services to third parties, expanding its portfolio of products and services and ultimately expanding their business opportunities.
  • It is maintained by media production professionals for media production professionals: As new codecs and media formats become available, you can bet that the open source implementations will be included in ACTIVAE 2.0!
